= Découvrez comment configurer l'authentification unique (SSO) dans StorageGRID à l'aide de SAML
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
La mise en place de l'authentification unique (SSO) dans StorageGRID permet à StorageGRID (le fournisseur de services) et au fournisseur d'identité SSO de communiquer de manière sécurisée au sujet des demandes d'authentification des utilisateurs.

Vous pouvez configurer l'authentification unique (SSO) dans StorageGRID à l'aide d'OpenID Connect (OIDC) ou de SAML. Suivez ces étapes générales pour configurer l'authentification unique dans StorageGRID à l'aide de SAML ou découvrez link:learn-about-setting-up-sso-oidc.html["Configuration de l'authentification unique (SSO) à l'aide d'OIDC"].

.image:https://raw.githubusercontent.com/NetAppDocs/common/main/media/number-1.png["Une seule"]Configurer l'authentification unique (SSO)
[role="quick-margin-para"]
Sélectionnez l'assistant de configuration SSO pour SAML. Configurez ensuite l'authentification unique avec votre fournisseur d'identité et passez en mode sandbox.

[role="quick-margin-para"]
Apprenez-en plus sur link:configure-sso-saml.html["Configuration de l'authentification unique (SSO) à l'aide de SAML"]

.image:https://raw.githubusercontent.com/NetAppDocs/common/main/media/number-2.png["Deux"]Configurer le fournisseur d'identité
[role="quick-margin-para"]
En fonction du fournisseur d'identité SSO que vous prévoyez d'utiliser, vous pouvez sélectionner Active Directory ou Entra ID pour le type de service LDAP de fédération d'identités.

[role="quick-margin-list"]
* Pour Active Directory Federation Service (AD FS), vous pouvez utiliser le fournisseur d'identité AD FS, Entra ID ou PingFederate.
* Pour Entra ID, vous ne pouvez utiliser que le fournisseur d'identité Entra ID.


[role="quick-margin-para"]
Utilisez le logiciel du fournisseur d'identité SSO pour configurer SAML pour chaque nœud d'administration de votre grille.

[role="quick-margin-list"]
* Découvrez link:creating-relying-party-trusts-in-ad-fs-saml.html["création d'un groupe d'applications pour chaque nœud d'administration"] dans Active Directory
* Découvrez link:creating-enterprise-application-entra-id-saml.html["création d'une application pour chaque nœud d'administration"] dans Entra ID
* Découvrez link:creating-sp-connection-ping-saml.html["création d'une connexion à un fournisseur de services pour chaque Admin Node"]dans PingFederate


.image:https://raw.githubusercontent.com/NetAppDocs/common/main/media/number-3.png["Trois"] Configuration de test
[role="quick-margin-para"]
Après avoir configuré SAML pour chaque nœud d'administration, retournez à l'assistant de configuration SSO et testez la configuration.

[role="quick-margin-para"]
Apprenez-en plus sur link:configure-sso-saml.html#test-sso["Test de la configuration pour l'authentification unique (SSO)"]

.image:https://raw.githubusercontent.com/NetAppDocs/common/main/media/number-4.png["Quatre"]Activer l’authentification unique (SSO)
[role="quick-margin-para"]
Activez l'authentification unique (SSO) pour tous les utilisateurs de StorageGRID.

[role="quick-margin-para"]
Apprenez-en plus sur link:configure-sso-saml.html#enable-sso-saml["Activation de l'authentification unique (SSO) pour les utilisateurs de StorageGRID"]
